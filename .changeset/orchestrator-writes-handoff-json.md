Fix the automatic handoff cycle never completing against a real agent. The orchestrator asked the agent to write `handoff.md`, then validated with `readHandoff()` — which also requires a schema-valid `handoff.json` that nothing ever wrote (the spec designates it machine-written). Against a real CLI this failed validation every time: the corrective retry told the agent "missing handoff.json", and the agent would fabricate one with the wrong schema, so the cycle fell into the non-destructive fallback and never respawned. (The fake-agent e2e masked it because the fake agent wrote a schema-correct `handoff.json` itself — something no real agent does.) The orchestrator now writes `handoff.json` itself after the agent produces a valid `handoff.md` — gathering `tool`, the real `sessionId` (from the turn-idle marker), `cwd`, git HEAD/dirty state, context %, and the previous-handoff chain link — then validates the pair. The fake agent now writes only `handoff.md`, so the e2e actually exercises this. Verified end-to-end against a real `claude` session: threshold → agent writes md → batonpass writes json → validate → kill → respawn → SessionStart injects the handoff.
